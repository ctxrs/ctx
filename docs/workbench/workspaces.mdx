## Overview

Workspaces represent repository roots in ctx.

A workspace is the boundary for settings, tasks, sessions, and review context.

## Workspace lifecycle

1. Add workspace
2. Open workspace
3. Configure baseline settings
4. Run tasks and sessions
5. Clean up stale entries when no longer needed

## Good workspace hygiene

- Add only true repo roots.
- Keep workspace names clear when managing multiple repos.
- Remove stale workspace entries you no longer use.

## Common pitfall

If behavior looks wrong, first verify you are in the expected workspace.

Many review and session issues are just "wrong workspace" mistakes.
