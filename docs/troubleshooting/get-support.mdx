Report issues at:

- https://github.com/ctxrs/ctx/issues
- support@ctx.rs for support requests that should not live in a public issue

## Legal and trust pages

- [Privacy Policy](/legal/privacy-policy)
- [Terms of Use](/legal/terms)
- [Security](/legal/security)

## Include this minimum info

- what you tried to do
- what happened
- what you expected
- exact error text
- OS and app version

## Helpful extras

- task/session context (without sensitive code)
- screenshots of relevant UI state
- whether issue reproduces after app restart

## Runtime startup details

For sandbox or runtime startup failures, include the execution mode and architecture. These details are especially useful for errors such as:

```text
linux sandbox ctx-mcp runtime is unavailable for aarch64
Details: start_failed
```

When reporting this class of issue, include:

- whether the workspace was configured as `Host` or `Sandbox`
- whether the machine is Apple Silicon / `aarch64`
- whether provider or harness downloads were skipped
- the most recent startup error from the ctx logs

If you need an immediate workaround while a sandbox-specific issue is being investigated, create a separate workspace in `Host` mode and keep the original sandbox workspace for debugging.
